### Что означает GPIO

GPIO — это **General Purpose Input/Output**, «вывод общего назначения». Это физическая ножка чипа, которой ты управляешь из кода: можешь подать на неё напряжение (режим _output_ — так зажигаются LED, пищит зуммер, щёлкает реле) или прочитать, есть ли на ней напряжение (режим _input_ — так читаются кнопка, PIR, ИК-датчик). Грубая аналогия из твоего мира: GPIO — это свойство, у которого есть и `get`, и `set`, только вместо значения в памяти на другом конце реальный провод с напряжением. Номер (GPIO 23) — это просто адрес конкретной ножки, по которому ты к ней обращаешься в коде.

Теперь по концепциям, которые стоят за твоей картой, — именно их новичку важно понять, чтобы ничего не спалить.

**Резистор 220 Ом на каждом LED.** Светодиод сам по себе ток не ограничивает — подключишь напрямую, и он потянет столько, что сгорит сам или просадит пин. Резистор последовательно (один вывод на GPIO, второй на ножку LED) гасит лишний ток. Поэтому у всех LED и каналов RGB в карте стоит «220 Ом».

**INPUT_PULLUP на кнопке.** Когда кнопка не нажата, пин «висит в воздухе» и читает случайный шум. Внутренняя подтяжка `INPUT_PULLUP` мягко притягивает пин к 3.3V, поэтому в покое он стабильно читает `HIGH` (1). Второй контакт кнопки идёт на GND, и при нажатии пин проваливается в `0`. То есть логика инвертированная: нажато = LOW. Это убирает нужду во внешнем резисторе.

**GPIO 34 и 35 — только вход (input-only) и ADC.** Ты выбрал их под потенциометр и фоторезистор очень грамотно. Это аналоговые входы канала ADC1, и они read-only — на них в принципе нельзя ничего «вывести», только измерять напряжение (0–4095 после `analogRead`). Важный нюанс: у пинов 34–39 нет внутренней подтяжки вообще, поэтому для аналогового чтения они идеальны, а для кнопки — нет (там и нужен был обычный пин с pull-up). И отдельно: ты не задействовал ADC2-пины под аналог — правильно, потому что ADC2 конфликтует с Wi-Fi, а у тебя проект завязан на Wi-Fi/MQTT.

**3.3V против 5V — самая частая ошибка новичков.** Логика ESP32 работает на 3.3V. PIR, ИК-модуль и плата реле в карте питаются от 5V (VIN), но это касается только их _питания_. Сигнал, который они отдают _обратно_ в GPIO, должен быть 3.3V-совместимым: HC-SR501 на выходе как раз даёт 3.3V — безопасно. А вот если поставишь датчик, который шлёт в пин полные 5V, нужен делитель напряжения или логический преобразователь, иначе можно повредить вход.

**Реле — активный LOW.** У большинства дешёвых модулей реле срабатывание происходит, когда на IN подаётся `LOW`, а не `HIGH`. То есть `digitalWrite(32, LOW)` включает реле, `HIGH` — выключает. Это контринтуитивно, поэтому держи в голове при написании логики.

**LCD 1602 — параллельный дисплей (без I2C).** У тебя символьный экран без синей платки-переходника, поэтому он подключается напрямую **6 сигнальными линиями**: RS (команда/данные), E (строб «прими данные») и 4 линии данных D4–D7. Это «4-битный режим»: каждый байт передаётся двумя порциями по 4 бита, чтобы сэкономить пины (иначе линий данных нужно 8). Плюс отдельный потенциометр на вывод **V0** — он задаёт контраст, без него экран пустой или в «квадратиках». Вывод **RW сажаем на GND** (только запись) — тогда 5В дисплея не попадёт обратно в GPIO ESP32. В отличие от I2C, каждая линия — это отдельный физический пин, шину на много устройств тут не построишь.

**И главное правило, без которого не заработает ничего:** все земли должны быть соединены. GND самой ESP32, GND макетной платы, GND внешнего 5V-питания (если питаешь PIR/реле отдельно) — всё это один общий минус. Если у компонента «своя» земля, не связанная с землёй ESP32, сигналы будут плавать и датчики начнут врать. На макетной плате это обычно одна общая минусовая шина, к которой приходит всё.

Отлично, теперь разберём саму планку — это ключевой момент, потому что breadboard соединяет дырки внутри себя по строгому правилу, и если его понять, расстановка станет очевидной. Сначала — как дырки связаны электрически.
![[Pasted image 20260618173442.png]]

Три правила, которые надо запомнить про эту планку:

Боковые полоски «+/−» — это шины питания. Все дырки внутри одной полоски соединены вдоль всей её длины. Их две пары (по бокам или сверху-снизу), и обычно их используют так: одну «−» делают общей землёй (GND), один «+» делают 3.3V, второй «+» — 5V. Маленький нюанс: на некоторых планках шина разорвана посередине — проверь мультиметром или просто кинь перемычку через весь разрыв, чтобы шина была сквозной.

Рабочая зона A–J работает иначе. В одном столбце (например, под цифрой 5) пять дырок `A B C D E` соединены между собой — это один электрический узел. Отдельно соединены `F G H I J` того же столбца — это второй узел. А между E и F идёт канавка, и она разрывает связь: верх и низ НЕ соединены. Соседние столбцы (5 и 6) тоже независимы.

Зачем нужна канавка — именно ради ESP32. Микросхемы и платы вроде ESP32 ставят «верхом» на канавку: левый ряд ножек попадает в верхнюю половину (A–E), правый ряд — в нижнюю (F–J). Если бы канавки не было, левые и правые ножки замкнулись бы между собой. А так каждая ножка ESP32 «выводится» в свой столбец из 5 дырок, и в оставшиеся 4 дырки этого столбца ты втыкаешь перемычки к компонентам.

## 1. Настройка проекта PlatformIO (делается один раз)

1. VS Code → иконка PlatformIO → **New Project**.

3. Открой `platformio.ini` и приведи к виду:

```ini
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
monitor_speed = 115200
lib_deps =
    adafruit/DHT sensor library
    adafruit/Adafruit Unified Sensor
    arduino-libraries/LiquidCrystal
```

4. Код каждого урока пиши в `src/main.cpp`.

# Блок A. Цифровой выход

## Урок 1 — Blink
![[Pasted image 20260618214834.png]]
Красный LED стоит ножками в `A6` (длинная нога, анод) и `A5` (короткая, катод). Резистор 220 Ω лежит между `B4` и `B5` — он соединяет столбец катода (5) со столбцом 4. Чёрная перемычка идёт из `A4` вверх в минусовую шину (GND). Зелёная перемычка идёт из `A6` (столбца анода) на вывод платы, помеченный `GPIO 23`.
 
**Схема:** красный LED длинной ногой (+) → резистор 220 Ом → GPIO 23; короткой ногой → GND.

```cpp
#include <Arduino.h>

#define LED_BUILTIN 2
#define LED_RED 23

void setup() {
  pinMode(LED_BUILTIN, OUTPUT);
  pinMode(LED_RED, OUTPUT);
}

void loop() {
  digitalWrite(LED_BUILTIN, HIGH);
  digitalWrite(LED_RED, HIGH);
  delay(500);
  digitalWrite(LED_BUILTIN, LOW);
  digitalWrite(LED_RED, LOW);
  delay(500);
}
```

**Проверка:** оба светодиода мигают раз в секунду.
**Концепты:** `pinMode`, `digitalWrite`, токоограничивающий резистор, полярность LED.

## Урок 2 — Светофор

**Схема:** добавь жёлтый (GPIO 19) и зелёный (GPIO 18) LED, каждый через 220 Ом.

```cpp
#include <Arduino.h>

#define LED_RED    23
#define LED_YELLOW 19
#define LED_GREEN  18

void setup() {
  pinMode(LED_RED, OUTPUT);
  pinMode(LED_YELLOW, OUTPUT);
  pinMode(LED_GREEN, OUTPUT);
}

void loop() {
  digitalWrite(LED_RED, HIGH);    delay(3000); digitalWrite(LED_RED, LOW);
  digitalWrite(LED_GREEN, HIGH);  delay(3000); digitalWrite(LED_GREEN, LOW);
  digitalWrite(LED_YELLOW, HIGH); delay(1000); digitalWrite(LED_YELLOW, LOW);
}
```

**Проверка:** красный → зелёный → жёлтый по кругу.
**Челлендж:** перепиши без `delay()`, на `millis()` (понадобится дальше для неблокирующего кода):

```cpp
unsigned long t = 0;
int phase = 0;
void loop() {
  unsigned long now = millis();
  if (phase == 0 && now - t > 3000) { /* переключить на зелёный */ phase = 1; t = now; }
  // ... и так далее
}
```

## Урок 3 — RGB-светодиод (PWM)

**Схема:** общий вывод RGB-светодиода — на GND (если **общий катод**) или на 3.3V (если **общий анод**). Три цветных вывода → каждый через 220 Ом → GPIO 25/26/27.

```cpp
#include <Arduino.h>

#define PIN_R 25
#define PIN_G 26
#define PIN_B 27
#define CH_R 0
#define CH_G 1
#define CH_B 2

void setColor(int r, int g, int b) {
  ledcWrite(CH_R, r);
  ledcWrite(CH_G, g);
  ledcWrite(CH_B, b);
}

void setup() {
  ledcSetup(CH_R, 5000, 8); ledcAttachPin(PIN_R, CH_R);
  ledcSetup(CH_G, 5000, 8); ledcAttachPin(PIN_G, CH_G);
  ledcSetup(CH_B, 5000, 8); ledcAttachPin(PIN_B, CH_B);
}

void loop() {
  setColor(255,0,0);   delay(700);  // красный
  setColor(0,255,0);   delay(700);  // зелёный
  setColor(0,0,255);   delay(700);  // синий
  setColor(255,255,0); delay(700);  // жёлтый
  setColor(0,255,255); delay(700);  // голубой
  setColor(255,0,255); delay(700);  // фиолетовый
}
```

> Если у тебя **arduino-esp32 3.x** — замени `setup()` на:
> ```cpp
> ledcAttach(PIN_R, 5000, 8);
> ledcAttach(PIN_G, 5000, 8);
> ledcAttach(PIN_B, 5000, 8);
> ```
> а в `setColor` пиши `ledcWrite(PIN_R, r)` (по пину, не по каналу).

**Проверка:** цвета сменяются плавно.
**Если цвета «инвертированы»** (255 = выключено) — у тебя общий анод: подай общий вывод на 3.3V и используй `255 - значение`.
**Концепты:** ШИМ (PWM), 8-битное разрешение (0–255), смешивание цветов.

## Урок 4 — Активный buzzer

**Схема:** «+» (длинная нога) → GPIO 13, «−» → GND.

```cpp
#include <Arduino.h>
#define BUZZER 13

void setup() { pinMode(BUZZER, OUTPUT); }

void loop() {
  digitalWrite(BUZZER, HIGH); delay(200);
  digitalWrite(BUZZER, LOW);  delay(800);
}
```

**Проверка:** короткий писк раз в секунду. Активному динамику достаточно постоянного `HIGH` — он сам генерирует тон.

## Урок 5 — Пассивный buzzer (мелодия)

**Схема:** на GPIO 14 и GND. Пассивному нужна **переменная частота** — генерим через `ledcWriteTone`.

```cpp
#include <Arduino.h>
#define BUZZER 14
#define CH 0

int melody[] = {262, 294, 330, 349, 392, 440, 494, 523}; // до ре ми фа соль ля си до

void setup() {
  ledcSetup(CH, 2000, 8);
  ledcAttachPin(BUZZER, CH);
}

void loop() {
  for (int i = 0; i < 8; i++) {
    ledcWriteTone(CH, melody[i]);
    delay(300);
  }
  ledcWriteTone(CH, 0); // тишина
  delay(1500);
}
```

**Проверка:** играет гамма. Разница с Уроком 4 теперь слышна: пассивный = ты сам задаёшь частоту.

# Блок C. Цифровой вход

## Урок 6 — Кнопка → Serial Monitor

**Схема:** один контакт кнопки → GPIO 4, второй → GND. Резистор не нужен — используем внутреннюю подтяжку.

```cpp
#include <Arduino.h>
#define BUTTON 4

int lastReading = HIGH;
unsigned long lastChange = 0;

void setup() {
  Serial.begin(115200);
  pinMode(BUTTON, INPUT_PULLUP); // не нажата = HIGH, нажата = LOW
}

void loop() {
  int reading = digitalRead(BUTTON);
  if (reading != lastReading && millis() - lastChange > 50) { // антидребезг 50 мс
    lastChange = millis();
    if (reading == LOW) {
      Serial.println("Кнопка нажата");
    }
    lastReading = reading;
  }
}
```

**Проверка:** в мониторе «Кнопка нажата» — ровно один раз на нажатие (а не 20). Это и есть **debounce**.
**Концепты:** `INPUT_PULLUP`, дребезг контактов.
![[Pasted image 20260621172008.png]]
---
Подвох с 4 контактами: на самом деле это не 4 независимые ноги, а 2 пары. Две ноги с одной стороны соединены между собой намертво (это один контакт), две с другой — второй контакт. Нажатие замыкает две пары между собой. То есть электрически у кнопки всего 2 вывода, просто каждый продублирован для механической устойчивости.

Отсюда главное правило подключения — ставь кнопку верхом на канавку. Тогда канавка проходит ровно между двумя парами: верхние две ноги попадают в верхнюю половину макетки, нижние — в нижнюю, и это гарантированно два разных узла. Дальше:

С верхней стороны бери одну ногу → перемычка на `GPIO4`. С нижней стороны одну ногу → перемычка в `GND`. Вторые ноги каждой пары можно не использовать (они для устойчивости).

Главная ошибка новичка: воткнуть две ноги с ОДНОЙ стороны канавки. Это две ноги одного контакта — они соединены всегда, и нажатие ничего не меняет (кнопка либо «всегда нажата», либо мертва). Если собрал, а реакции нет — поверни кнопку на 90°.

Как убедиться, если сомневаешься: мультиметр в прозвонке. Две ноги, которые пищат БЕЗ нажатия — это одна пара (один контакт), их вместе использовать нельзя. А пара, которая пищит только при нажатии, — твоя рабочая, по одной ноге из неё на `GPIO4` и `GND`.

Твой код из урока 6 правильный и менять его не нужно — `INPUT_PULLUP` + антидребезг 50 мс. Собери кнопку как на схеме, открой Serial Monitor на 115200 и жми — должно падать «Кнопка нажата» по разу на нажатие.


`Serial.begin(115200);` — это инициализация последовательного порта (UART) для связи платы с компьютером по USB. Вызывается один раз в `setup()`.

Что он делает: открывает «канал», по которому ESP32 шлёт и принимает текст через USB. Пока ты его не вызвал, `Serial.println("Кнопка нажата")` ничего не выведет — писать некуда, поток не открыт. Аналогия из твоего мира: это как инициализировать логгер/`StreamWriter` перед тем, как в него писать. `Serial.begin()` открывает поток, `Serial.println()` пишет в него.

Число `115200` — скорость в бодах (бит/с). Ключевое правило: оно должно совпадать с настройкой Serial Monitor, то есть с тем самым `monitor_speed = 115200` в `platformio.ini`. Если не совпадёт — в мониторе будут кракозябры вместо текста, потому что обе стороны должны «договориться» о скорости.

Так что в твоём уроке 6 цепочка такая: `Serial.begin(115200)` в `setup()` открыл канал → при нажатии кнопки `Serial.println` шлёт строку → ты открываешь Serial Monitor на 115200 и видишь «Кнопка нажата».

Открой монитор (кнопка с вилкой внизу PlatformIO или `pio device monitor`), нажми кнопку — и проверь, что строки идут по одной на нажатие.
## Урок 7 — Кнопка-переключатель (toggle)

**Схема:** та же кнопка (GPIO 4) + красный LED (GPIO 23).

```cpp
#include <Arduino.h>
#define BUTTON 4
#define LED_RED 23

bool ledOn = false;
int lastReading = HIGH;
unsigned long lastChange = 0;

void setup() {
  pinMode(BUTTON, INPUT_PULLUP);
  pinMode(LED_RED, OUTPUT);
}

void loop() {
  int reading = digitalRead(BUTTON);
  if (reading == LOW && lastReading == HIGH && millis() - lastChange > 50) {
    ledOn = !ledOn;                              // переключаем состояние
    digitalWrite(LED_RED, ledOn ? HIGH : LOW);
    lastChange = millis();
  }
  lastReading = reading;
}
```

**Проверка:** одно нажатие включает LED, следующее выключает.
**Концепт:** детект **фронта** нажатия (HIGH→LOW), а не уровня.

## Урок 8 — Датчик движения PIR (HC-SR501)

**Схема:** VCC → **5V (VIN)** Important need 5V, GND → GND, OUT → GPIO 5. Реакция: красный LED + активный buzzer.

```cpp
#include <Arduino.h>
#define PIR 5
#define LED_RED 23
#define BUZZER 13

void setup() {
  Serial.begin(115200);
  pinMode(PIR, INPUT);
  pinMode(LED_RED, OUTPUT);

}

void loop() {
  if (digitalRead(PIR) == HIGH) {
    Serial.println("Движение!");
    digitalWrite(LED_RED, HIGH);
    digitalWrite(BUZZER, HIGH);
  } else {
    digitalWrite(LED_RED, LOW);
    digitalWrite(BUZZER, LOW);
  }
  delay(100);
}
```

```cpp
#include <Arduino.h>
```

# Блок D. Аналоговый вход (ADC)

## Урок 10 — Потенциометр → яркость LED

**Схема:** крайние выводы потенциометра → 3.3V и GND, средний → GPIO 34. Красный LED — на GPIO 23 (через PWM).

```cpp
#include <Arduino.h>
#define POT 34
#define LED_RED 23
#define CH 0

void setup() {
  Serial.begin(115200);
  ledcSetup(CH, 5000, 8);
  ledcAttachPin(LED_RED, CH);
}

void loop() {
  int raw = analogRead(POT);          // 0..4095 (12 бит)
  int duty = map(raw, 0, 4095, 0, 255);
  ledcWrite(CH, duty);
  Serial.println(raw);
  delay(50);
}
```

**Проверка:** крутишь ручку — меняется яркость; в мониторе бежит число 0–4095.
**Концепты:** `analogRead`, 12-битный ADC, `map()`, связка ADC → PWM.

## Урок 11 — Фоторезистор (автоматический ночник)

**Схема:** модуль фоторезистора: VCC → 3.3V, GND → GND, **A0 → GPIO 35**. LED — GPIO 23.

```cpp
#include <Arduino.h>
#define LDR 35
#define LED_RED 23

int threshold = 2000; // подбери под свою комнату по выводу в мониторе

void setup() {
  Serial.begin(115200);
  pinMode(LED_RED, OUTPUT);
}

void loop() {
  int light = analogRead(LDR);
  Serial.println(light);
  if (light < threshold) {        // темно
    digitalWrite(LED_RED, HIGH);
  } else {
    digitalWrite(LED_RED, LOW);
  }
  delay(100);
}
```

**Калибровка:** закрой датчик ладонью, посмотри число; освети — посмотри снова; поставь `threshold` между ними.
> Если на модуле нет пина A0, а только D0 — тогда это цифровой вход как в Уроке 9 (порог крутится потенциометром на модуле).

# Блок E. Датчики данных

## Урок 12 — DHT11 (температура + влажность)

**Схема:** VCC → 3.3V, GND → GND, DATA → GPIO 16.

```cpp
#include <Arduino.h>
#include <DHT.h>

#define DHTPIN 16
#define DHTTYPE DHT11
```

# Блок F. Дисплей (LCD 1602 — параллельный, без I2C)

## Урок 13 — LCD 1602 «Hello ESP32»

**Схема (4-битный режим, без I2C):** VSS→GND, VDD→**5V (VIN)**, V0→средний вывод потенциометра (контраст), RS→GPIO 13, RW→GND, E→GPIO 12, D4→GPIO 27, D5→GPIO 26, D6→GPIO 25, D7→GPIO 34, A→5V через 220 Ом, K→GND. Контакты D0–D3 (7–10) в 4-битном режиме не используются.

```
  LCD 1602 (16 конт.)     ESP32
  ──────────────────     ─────
   1 VSS ─────────────── GND
   2 VDD ─────────────── 5V (VIN)
   3 V0  ── движок потенциометра (контраст)
   4 RS  ─────────────── GPIO 13
   5 RW  ─────────────── GND
   6 E   ─────────────── GPIO 12
  11 D4  ─────────────── GPIO 27
  12 D5  ─────────────── GPIO 26
  13 D6  ─────────────── GPIO 25
  14 D7  ─────────────── GPIO 34
  15 A   ── 5V через 220 Ом   (подсветка +)
  16 K   ─────────────── GND  (подсветка −)
```

```cpp
#include <Arduino.h>
#include <LiquidCrystal.h>

//              RS  E   D4  D5  D6  D7
LiquidCrystal lcd(13, 12, 27, 26, 27, 17);

void setup() {
  lcd.begin(16, 2);              // 16 символов, 2 строки
  lcd.print("Hello ESP32");
}

void loop() {
  lcd.setCursor(0, 1);           // столбец 0, вторая строка
  lcd.print(millis() / 1000);    // секунды от старта
}
```

> **Питание и уровни.** VDD дисплея — от **5V (VIN)**, иначе экран тусклый/пустой. ESP32 отдаёт на GPIO 3.3V — для HD44780 при питании 5V это обычно работает; если символы «пляшут», запитай VDD от 3.3V (будет бледнее, зато логические уровни совпадут). **RW обязательно на GND** — тогда 5В не попадёт в пины ESP32.
> Если экран засветился, но пустой (или видны «квадратики») — **крути потенциометр контраста на V0**. Это самая частая ошибка новичка.
> В уроках дисплея (13, 14, 18) RGB-светодиод и ИК-датчик не подключены — их пины 25/26/27 и 17 заняты дисплеем.
**Концепты:** параллельный дисплей, 4-битная шина (RS/E/D4–D7), контраст через потенциометр.

## Урок 14 — LCD 1602 + DHT11

**Схема:** объедини схемы уроков 12 (DHT11 на GPIO 16) и 13 (LCD).

```cpp
#include <Arduino.h>
#include <LiquidCrystal.h>
#include <DHT.h>

//              RS  E   D4  D5  D6  D7
LiquidCrystal lcd(21, 22, 25, 26, 27, 17);

#define DHTPIN 16
#define DHTTYPE DHT11
DHT dht(DHTPIN, DHTTYPE);

void setup() {
  lcd.begin(16, 2);
  dht.begin();
}

void loop() {
  float t = dht.readTemperature();
  float h = dht.readHumidity();

  lcd.setCursor(0, 0);
  lcd.print("T: ");
  lcd.print(t, 1);            // 1 знак после точки
  lcd.print(" C   ");         // пробелы затирают старые цифры

  lcd.setCursor(0, 1);
  lcd.print("H: ");
  lcd.print(h, 0);
  lcd.print(" %   ");

  delay(2000);
}
```

**Проверка:** температура и влажность на экране, обновляются каждые 2 сек. Это твой первый «прибор».
> `LiquidCrystal` не умеет `printf` (в отличие от OLED-библиотеки). Числа выводим через `lcd.print(t, 1)` — второй аргумент это число знаков после запятой. Хвостовые пробелы `"   "` затирают старые символы: без них «25.0» после «5.0» оставит лишнюю цифру-призрак.

# Блок G. Реле

## Урок 15 — Реле: одноканальное + двухканальное (3 канала)

Собираем в одной схеме **оба** модуля, которые есть в наборах:
- **двухканальное реле** (из набора ESP32) — 2 канала;
- **одноканальное реле** (из набора Arduino Uno) — 1 канал.

Итого 3 независимых канала. Управлять будем **неблокирующим** циклом (`millis()`, без `delay()`) — так, как это делают в реальном роботе, где процессор одновременно опрашивает Wi-Fi, кнопки и датчики.

**Схема:**

| Модуль        | Вывод | → ESP32      |
| ------------- | ----- | ------------ |
| Двухканальное | VCC   | **5V (VIN)** |
| Двухканальное | GND   | GND          |
| Двухканальное | IN1   | GPIO 32      |
| Двухканальное | IN2   | GPIO 33      |
| Одноканальное | VCC   | **5V (VIN)** |
| Одноканальное | GND   | GND          |
| Одноканальное | IN    | GPIO 25      |

> ⚠️ **Общая земля.** Оба модуля и ESP32 сидят на одном GND — иначе сигнал IN «не имеет точки отсчёта» и реле щёлкает хаотично. Питание +5V берём с вывода **VIN** платы (когда она запитана по USB, там ~5V).

```cpp
#include <Arduino.h>

// Двухканальное реле (набор ESP32)
#define RELAY_2CH_1 32
#define RELAY_2CH_2 33
// Одноканальное реле (набор Arduino Uno)
#define RELAY_1CH   25

// Активный LOW: помощники, чтобы не путаться с уровнями
inline void relayOn(uint8_t pin)  { digitalWrite(pin, LOW);  } // LOW = включено
inline void relayOff(uint8_t pin) { digitalWrite(pin, HIGH); } // HIGH = выключено

const uint8_t RELAYS[3] = { RELAY_2CH_1, RELAY_2CH_2, RELAY_1CH };
const char*   NAMES[3]  = { "Канал 1 (2ch)", "Канал 2 (2ch)", "Реле (1ch)" };

const unsigned long STEP_MS = 1500; // длительность одного шага
unsigned long lastStep = 0;
int stage = 0;

void allOff() {
  for (int i = 0; i < 3; i++) relayOff(RELAYS[i]);
}

void setup() {
  Serial.begin(115200);
  for (int i = 0; i < 3; i++) pinMode(RELAYS[i], OUTPUT);
  allOff();                     // при старте всё выключено
  Serial.println("Реле готовы");
}

void loop() {
  if (millis() - lastStep >= STEP_MS) {
    lastStep = millis();
    allOff();                   // гасим предыдущий канал
    relayOn(RELAYS[stage]);     // включаем текущий
    Serial.printf("ON: %s\n", NAMES[stage]);
    stage = (stage + 1) % 3;    // по кругу: 0 → 1 → 2 → 0
  }
  // loop() не заблокирован delay() — сюда можно добавить опрос кнопки,
  // Wi-Fi, датчиков. Это задел на робота из Этапа 2.
}
```

**Проверка:** каналы включаются по кругу раз в 1.5 сек — сначала два щелчка на двухканальном модуле, потом один на одноканальном. В Serial Monitor (115200) видно, какой канал включён. На каждом модуле загорается светодиод активного канала.

> **Активный низкий уровень:** `LOW` = реле включено, `HIGH` = выключено. Чтобы не путаться, завёл функции `relayOn()` / `relayOff()` — код читается по смыслу, а не по уровням.
> **3.3 В против 5 В.** Модуль из Uno-набора рассчитан на 5В-логику, а ESP32 выдаёт на GPIO **3.3 В**. Обычно этого хватает, чтобы реле сработало. Если одноканальное щёлкает нестабильно или не срабатывает — это нехватка уровня: питай модуль от 5V (как в схеме), перемычку **JD-VCC** не снимай, а при упорной проблеме поставь между GPIO и IN транзистор-ключ или модуль с оптронным входом на 3.3В.
> **Почему `millis()`, а не `delay()`.** `delay(1000)` замораживает весь процессор — робот в это время «слепой». Схема на `millis()` проверяет время и сразу отпускает `loop()`, поэтому параллельно можно опрашивать сеть и датчики. Это базовый паттерн всей встраиваемой разработки.
> Реальную нагрузку 220В на этом этапе **не подключай** — учимся на щелчках и светодиодах.

**Мини-задание:** сделай так, чтобы каналы не гасли по очереди, а **накапливались** (сначала горит один, потом два, потом три, затем все гаснут и цикл повторяется). Подсказка: убери `allOff()` из шага и гаси всё только когда `stage` вернулся в 0.

## Урок 17 — HTTP-сервер (данные + управление)

**Схема:** DHT11 (GPIO 16) + реле (GPIO 32). Страница показывает температуру и даёт кнопки вкл/выкл реле.

```cpp
#include <Arduino.h>
#include <WiFi.h>
#include <WebServer.h>
#include <DHT.h>

const char* SSID = "ТВОЙ_WIFI";
const char* PASS = "ПАРОЛЬ";

#define DHTPIN 16
#define DHTTYPE DHT11
#define RELAY1 32

DHT dht(DHTPIN, DHTTYPE);
WebServer server(80);

void handleRoot() {
  float t = dht.readTemperature();
  String html = "<h1>ESP32</h1>";
  html += "<p>Температура: " + String(t, 1) + " C</p>";
  html += "<a href='/on'>Включить реле</a><br><a href='/off'>Выключить реле</a>";
  server.send(200, "text/html; charset=utf-8", html);
}

void setup() {
  Serial.begin(115200);
  pinMode(RELAY1, OUTPUT);
  digitalWrite(RELAY1, HIGH);
  dht.begin();

  WiFi.begin(SSID, PASS);
  while (WiFi.status() != WL_CONNECTED) { delay(500); }
  Serial.println(WiFi.localIP());

  server.on("/", handleRoot);
  server.on("/on",  []() { digitalWrite(RELAY1, LOW);  server.send(200, "text/plain", "ON"); });
  server.on("/off", []() { digitalWrite(RELAY1, HIGH); server.send(200, "text/plain", "OFF"); });
  server.begin();
}

void loop() {
  server.handleClient();
}
```

**Проверка:** открой `http://<IP-из-монитора>/` в браузере телефона/ПК (в той же сети) — видишь температуру и кнопки, реле щёлкает.

## Урок 18 — Капстоун: мини-метеостанция

Объединяем **всё**: датчики (DHT11 + фоторезистор) → выводим локально на **LCD 1602** и отдаём по сети как **JSON**.

> У 1602 всего 2 строки, а показать надо 4 значения — поэтому экран **чередует две страницы** каждые 2 секунды: сначала температура+влажность+свет, потом IP-адрес.

```cpp
#include <Arduino.h>
#include <WiFi.h>
#include <WebServer.h>
#include <LiquidCrystal.h>
#include <DHT.h>

const char* SSID = "ТВОЙ_WIFI";
const char* PASS = "ПАРОЛЬ";

//              RS  E   D4  D5  D6  D7
LiquidCrystal lcd(21, 22, 25, 26, 27, 17);

#define DHTPIN 16
#define DHTTYPE DHT11
#define LDR 35

DHT dht(DHTPIN, DHTTYPE);
WebServer server(80);

bool page = false;   // false = датчики, true = IP

void handleData() {
  float t = dht.readTemperature();
  float h = dht.readHumidity();
  int light = analogRead(LDR);
  String json = "{";
  json += "\"temperature\":" + String(t, 1) + ",";
  json += "\"humidity\":"    + String(h, 0) + ",";
  json += "\"light\":"       + String(light);
  json += "}";
  server.send(200, "application/json", json);
}

void setup() {
  Serial.begin(115200);
  lcd.begin(16, 2);
  dht.begin();

  WiFi.begin(SSID, PASS);
  while (WiFi.status() != WL_CONNECTED) { delay(500); }
  Serial.println(WiFi.localIP());

  server.on("/data", handleData);
  server.begin();
}

void loop() {
  server.handleClient();

  float t = dht.readTemperature();
  float h = dht.readHumidity();
  int light = analogRead(LDR);

  lcd.clear();
  if (!page) {
    // страница 1: T / H / свет
    lcd.setCursor(0, 0);
    lcd.print("T:"); lcd.print(t, 1);
    lcd.print(" H:"); lcd.print(h, 0); lcd.print("%");
    lcd.setCursor(0, 1);
    lcd.print("Light: "); lcd.print(light);
  } else {
    // страница 2: IP-адрес
    lcd.setCursor(0, 0);
    lcd.print("IP address:");
    lcd.setCursor(0, 1);
    lcd.print(WiFi.localIP());   // IPAddress печатается напрямую
  }
  page = !page;

  delay(2000);
}
```

**Проверка:**
- на LCD по очереди: страница «T/H/свет» и страница с IP-адресом (сменяются каждые 2 сек, т.к. в 2 строки всё сразу не влезает);
- `http://<IP>/data` отдаёт JSON `{"temperature":24.0,"humidity":40,"light":1500}`.

> **Это и есть мост к [[ESP32 - продвинутый гайд|Этапу 2]]:** тот же JSON ты дальше будешь слать не по HTTP, а по **MQTT → RabbitMQ → .NET Worker → PostgreSQL**. Код датчиков останется почти тем же — поменяется только транспорт.

## Частые проблемы

| Симптом | Причина / решение |
|---|---|
| Не заливается прошивка (`Connecting......`) | Зажми кнопку **BOOT** на плате в момент старта заливки |
| Порт COM не виден | Драйвер CP210x/CH340 — см. [[Install Arduino driver]]; проверь Device Manager |
| Кракозябры в мониторе | `monitor_speed = 115200` в `platformio.ini` и та же скорость в мониторе |
| PIR/реле/ИК работают через раз | Питание не от 5V, а от 3.3V — переключи на `VIN`/`5V` |
| LCD светится, но пустой / «квадратики» | Крути потенциометр **контраста на V0**; проверь RS/E и D4–D7 |
